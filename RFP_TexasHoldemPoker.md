## 3. 参加者数に応じたゲームロジック

### 3.1 参加者管理システム

#### 3.1.1 基本ルール
- **人の参加者が1-4人の場合：**
  - 総参加者数を4人に固定
  - 不足分はAI（PC）プレイヤーで補完
  - 例：人2人 + AI2人 = 4人でゲーム実行

- **人の参加者が5-10人の場合：**
  - AIプレイヤーは参加しない
  - 人のプレイヤーのみでゲーム実行
  - 最大10人まで参加可能

#### 3.1.2 AI参加判定ロジック

```
IF 人の参加者数 <= 4 THEN
    AI参加者数 = 4 - 人の参加者数
    総参加者数 = 4
ELSE IF 人の参加者数 >= 5 AND 人の参加者数 <= 10 THEN
    AI参加者数 = 0
    総参加者数 = 人の参加者数
ELSE
    エラー: 参加者数上限超過
END IF
```

### 3.2 AI プレイヤー仕様

#### 3.2.1 難易度レベル
- **初級AI：** 保守的なプレイスタイル、基本的な判断のみ
- **中級AI：** バランスの取れたプレイ、オッズ計算を考慮
- **上級AI：** 攻撃的なプレイ、心理戦を含む高度な判断

#### 3.2.2 人間らしさの実装
- ランダムな思考時間（1-5秒）
- 時々の非合理的な判断
- プレイスタイルの個性化
- 疲労による判断力の変化

#### 3.2.3 AI識別表示
- プレイヤー名の後に「(AI)」表示
- AIレベルの表示（初級/中級/上級）
- 統計情報での人間とAIの区別

## 5. ゲームフロー

### 5.1 ゲーム開始プロセス
1. **参加者集合**：ルームに参加者が集まる
2. **参加者数判定**：人の参加者数をチェック
3. **AI追加判定**：4人以下の場合、AIプレイヤーを追加

3. **プリフロップ**：第1ベッティングラウンド
4. **フロップ**：コミュニティカード3枚オープン
5. **ターン**：4枚目のカードオープン
6. **リバー**：5枚目のカードオープン
7. **ショーダウン**：勝敗決定
